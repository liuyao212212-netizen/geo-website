## BlurText首次不显示 + 团队排版错乱修复
- **问题1**：分页面副标题（如"GEO 白皮书"等BlurText组件）首次进入页面时不显示，需刷新多次才出现
  - **根因**：motion-v的Motion组件在路由切换时，animate prop从`fromSnapshot`快速变为keyframes对象，Motion动画引擎可能错过关键帧动画触发时机（竞态条件）
  - **修复**：BlurText.vue添加`ready` ref状态机
    - 只有在`inView`确定后才渲染`<Motion>`组件，避免挂载时animate prop变化导致动画错过
    - `ready`前渲染不可见占位文本（opacity:0; filter:blur(10px)），保持DOM布局稳定
    - IntersectionObserver触发时同时设置`inView=true` + `ready=true`
    - 600ms后备定时器：即使Observer未触发也强制ready，确保Motion至少能渲染
    - `onActivated` hook也重置状态并重新检测（支持keep-alive场景）
- **问题2**：手机端核心团队板块排版错乱，卡片挤在一起无间距（iOS Safari截图）
  - **根因**：iOS Safari中`grid` + `perspective` + `transform-style:preserve-3d` + `backface-visibility`组合已知会导致`grid-gap`失效
  - **修复**：AboutView.vue的`.team-grid`在`@media (max-width: 640px)`下从grid改为`flex-direction:column`
    - `display:flex; flex-direction:column; align-items:center; gap:$space-6`
    - `.flip-card`在移动端添加`width:100%; max-width:360px`，确保flex子元素不超出容器
